## 一、前端切分步驟

### 商戶一（葡眾）— 現有專案

| 項目 | 值 |
|------|-----|
| Vercel Project | 已存在 |
| Build Command | `yarn build:propartner` |
| Output Directory | `dist` |
| 環境變數 | `VITE_API_BASE` = `https://propartnerbe.vercel.app` |

**不需任何變更**，推送到 main 即自動部署。

### 商戶二 — 新建 Vercel 專案

1. Vercel → **Add New Project** → 匯入同一個 `propartner` GitHub repo
2. 設定：

| 項目 | 值 |
|------|-----|
| Build Command | `yarn build:merchant2` |
| Output Directory | `dist` |
| 環境變數 | `VITE_API_BASE` = `https://<merchant2-backend>.vercel.app` |

3. 部署後會得到獨立的前端網址（如 `merchant2.vercel.app`）

### 新增更多商戶

```bash
# 1. 建立 .env 檔
cp .env.merchant2 .env.merchant3
# 編輯 VITE_API_BASE 指向新後端

# 2. package.json 加指令
"dev:merchant3": "vite --mode merchant3",
"build:merchant3": "vite build --mode merchant3"

# 3. Vercel 新增 Project，Build Command = yarn build:merchant3
```

## 二、後端切分步驟

### 核心原則：同一份程式碼，不同 DB 連線

後端 repo (`ledbackend`) 目前 **DB 連線字串寫死在 `config/dbConnection.js`**，上線前須改為環境變數。

### 必做：改用環境變數連線 DB

**`config/dbConnection.js`** 需改為：

```javascript
const mongoose = require("mongoose")

const connectDb = async () => {
    try {
        const connect = await mongoose.connect(process.env.DATABASE_URL)
        console.log('DB connected', connect.connection.host, connect.connection.name)
    } catch (err) {
        console.log(err)
        process.exit(1)
    }
}

module.exports = connectDb
```

### 商戶一（葡眾）後端

| 項目 | 值 |
|------|-----|
| Vercel Project | `propartnerbe`（已存在） |
| 環境變數 | `DATABASE_URL` = `mongodb+srv://.../<propartner-db-name>?retryWrites=true&w=majority` |
| 環境變數 | `DRIVE_KEY` = （現有 Google Drive 金鑰） |

### 商戶二後端

1. Vercel → **Add New Project** → 匯入同一個 `ledbackend` GitHub repo
2. 設定：

| 項目 | 值 |
|------|-----|
| Build Command | （Vercel 自動偵測） |
| 環境變數 | `DATABASE_URL` = `mongodb+srv://.../<merchant2-db-name>?retryWrites=true&w=majority` |
| 環境變數 | `DRIVE_KEY` = （如需照片上傳功能再設定） |

3. 部署完成後取得後端網址，填入前端的 `.env.merchant2`

## 三、MongoDB 資料庫切分

### 方案 A：同一個 Atlas Cluster，不同 Database（推薦）

```
Cluster0 (atlas)
├── propartner-db      ← 商戶一的所有 collections
└── merchant2-db       ← 商戶二的所有 collections
```

- 連線字串只需改 database name：
  - 葡眾：`mongodb+srv://user:pass@cluster0.xxx.mongodb.net/propartner-db`
  - 商戶二：`mongodb+srv://user:pass@cluster0.xxx.mongodb.net/merchant2-db`
- 優點：免費方案即可，管理簡單
- 缺點：共享 Cluster 資源

### 方案 B：完全獨立 Cluster

- 各商戶各自一個 Atlas Cluster
- 優點：完全隔離
- 缺點：成本較高

- [ ] 確認 `index.js` 首頁路由不要外洩 `DRIVE_KEY`（目前 `res.send(... + process.env.DRIVE_KEY)`）

## 五、已知需注意事項

| 問題 | 影響 | 建議 |
|------|------|------|
| `Num` model 用 hardcoded `_id: '65ddb64d492e821995a3c319'` 查詢 | 新 DB 不會有這筆資料 | 每個新 DB 需手動建立初始 Num 文件，或改用 `findOne()` 不指定 `_id` |
| `Time` model 同上 hardcoded `_id` | 同上 | 同上 |
| `GET /` 回應包含 `DRIVE_KEY` | 資安風險 | 移除或改為安全的健康檢查回應 |
| Settings 密碼 `caq6866` 寫死在前端 | 所有商戶共用同一密碼 | 可考慮改為 env 變數或後端驗證 |

## 六、部署流程圖

```
1. 修改後端 DB 連線 → 推送 ledbackend repo
         │
2. Vercel 自動部署商戶一後端（propartnerbe）
         │
3. Vercel 手動建立商戶二後端 Project → 設定 DATABASE_URL → 部署
         │
4. 取得商戶二後端網址 → 寫入 .env.merchant2
         │
5. 推送 propartner repo
         │
6. Vercel 自動部署商戶一前端
         │
7. Vercel 手動建立商戶二前端 Project → 設定 Build Command + VITE_API_BASE → 部署
         │
8. 驗證兩個商戶各自獨立運作 ✅
```
